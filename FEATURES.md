# FEATURES.md — Предложения по развитию FileInfo

## Текущее состояние

FileInfo — утилита для просмотра и редактирования свойств файлов Windows: временны́е метки, атрибуты, NTFS-потоки, EXIF-метаданные, уникальный ID файла. Имеет интеграцию с контекстным меню Explorer. Ниже — конкретные предложения с приоритетами.

---

## Новая функциональность

### 1. Просмотр содержимого потока (Streams → View)
**Приоритет: высокий**

Кнопка «Просмотреть» сейчас показывает только имя потока в MessageBox. Нужно открыть содержимое:
- Для текстовых данных — отобразить в окне редактора (ReadOnly Edit или Scintilla-подобный контрол).
- Для бинарных — показать hex-дамп (16 байт в строке, классический формат xxd).
- Автоопределение: если данные валидный UTF-8/UTF-16 — текстовый режим, иначе — hex.
- Ограничение предпросмотра: первые 64 KB, кнопка «Сохранить как...» для полного содержимого.

### 2. Удаление потока (Streams → Delete)
**Приоритет: высокий**

Отсутствует базовая операция. Реализуется через `DeleteFile` на имени `file.ext:stream_name`. Добавить:
- Пункт в контекстное меню ListView.
- Подтверждение удаления с именем потока.
- Обновление списка после удаления.

### 3. Редактирование EXIF/IPTC/XMP метаданных
**Приоритет: средний**

Вкладка EXIF сейчас только для чтения. Exiv2 поддерживает запись. Добавить:
- Двойной клик по строке → инлайн-редактор значения.
- Кнопки «Сохранить изменения» / «Отменить» в нижней части вкладки.
- Запрет редактирования полей только для чтения (тип `Undefined`, бинарные теги).

### 4. Копирование данных в буфер обмена
**Приоритет: средний**

Во всех ListView-таблицах (Streams, EXIF, ID) нет возможности скопировать данные:
- `Ctrl+C` — копирует выделенные строки как текст (Tab-разделённые столбцы).
- `Ctrl+A` — выделить всё.
- Пункт «Копировать» в контекстном меню каждой таблицы.

### 5. Дополнительные вкладки из FILE_INFO_BY_HANDLE_CLASS
**Приоритет: средний**

В коде закомментированы 7 классов, данные которых полезны:

| Класс | Что показывает |
|-------|----------------|
| `FileCompressionInfo` | Сжатый размер, метод сжатия |
| `FileAttributeTagInfo` | ReparsePoint тег (тип симлинка, junction и т.д.) |
| `FileStorageInfo` | Логический / физический размер сектора |
| `FileAlignmentInfo` | Требование выравнивания буфера |

Их можно добавить как подразделы на существующих вкладках (Basic или Standard) без создания новых табов.

### 6. Поддержка длинных путей (>260 символов)
**Приоритет: средний**

Во всём коде используется `MAX_PATH`. Windows 10 1607+ поддерживает пути до 32 767 символов через `\\?\`-префикс. Нужно:
- При открытии файла добавлять `\\?\` если путь > 260 символов.
- Заменить `TCHAR[MAX_PATH]` буферы на динамические `std::wstring` / `HeapAlloc`.

### 7. Перетаскивание файла на иконку в панели задач
**Приоритет: низкий**

Расширить D&D: сейчас работает только перетаскивание на окно. Добавить `IDropTarget` на иконку в Taskbar через `ITaskbarList3`.

### 8. Последние открытые файлы
**Приоритет: низкий**

История последних 10 файлов в системном меню окна (`GetSystemMenu`) или в выпадающем списке рядом с полем пути. Хранить в реестре `HKCU\Software\FileInfo\RecentFiles`.

### 9. Сравнение свойств двух файлов
**Приоритет: низкий**

Второй экземпляр приложения или режим «Сравнить с...»: открыть второй файл и показать различия атрибутов и временны́х меток рядом.

## Доработка имеющегося

### 10. Форматирование чисел в Standard Info
**Приоритет: средний**

Поля «Allocated Size» и «End of File» выводят сырые числа без разделителей. Использовать `GetNumberFormat` / `StrFormatByteSize` для отображения в виде «1 234 567 890 байт (1,15 ГБ)».

### 11. Timezone/DST при работе с временны́ми метками
**Приоритет: средний**

`FileTimeToLocalFileTime` не учитывает сдвиг DST корректно для исторических дат. Использовать `TzSpecificLocalTimeToSystemTime` / `SystemTimeToTzSpecificLocalTime`. Добавить индикатор часового пояса рядом с полями дат.

### 12. Подтверждение перед сохранением временны́х меток
**Приоритет: средний**

Кнопка «Сохранить» применяет изменения без вопросов. Добавить диалог «Сохранить изменения временны́х меток для файла X?» аналогично предупреждению readonly.

### 13. Валидация имени потока при создании
**Приоритет: средний**

Инлайн-редактор при создании потока проверяет только пустую строку. Запрещённые символы в именах ADS: `:`, `/`, `\`, `*`, `?`, `"`, `<`, `>`, `|`, а также имя не должно начинаться с `$`. Добавить проверку и подсветку ошибки.

### 14. Кнопка «Обновить» (Refresh)
**Приоритет: средний**

Нет способа перечитать данные файла без его повторного открытия. Добавить:
- Кнопку «Обновить» или клавишу `F5`.
- Автоматическое обновление при изменении файла (через `ReadDirectoryChangesW` или `FindFirstChangeNotification`).

### 15. Атрибут ReparsePoint
**Приоритет: низкий**

Чекбокс `ReparsePoint` (IDC_CHECK12) есть в ресурсах, но в коде не используется. Подключить: если установлен `FILE_ATTRIBUTE_REPARSE_POINT`, показать чекбокс включённым и отобразить тип (симлинк / junction / mount point) через `FileAttributeTagInfo`.

### 16. Отображение владельца файла
**Приоритет: низкий**

В Basic Info добавить поле «Владелец» (Owner) — читать через `GetSecurityInfo` + `LookupAccountSid`. Только отображение, без редактирования.

### 17. Отображение размера файла в Standard Info
**Приоритет: низкий**

Поля «Выделено» и «Конец файла» дублируют данные из проводника без контекста. Добавить «Степень сжатия NTFS» = `Allocated / EndOfFile * 100%` при наличии атрибута Compressed.

### 18. Прогресс при записи/чтении больших потоков
**Приоритет: низкий**

Операции чтения/записи ADS блокируют UI. Перенести их в отдельный поток (`CreateThread`) с прогресс-диалогом (`IProgressDialog` из Shell API).

## Технические долги

### 19. Поддержка MAX_PATH в диалоге открытия файла
Заменить `SHBrowseForFolder` на `IFileOpenDialog` (Vista+ API) — поддерживает длинные пути, фильтрацию, предпросмотр, множественный выбор.

### 20. Иконка приложения
Нет иконки самого приложения (только `IDI_FILEOPEN` для кнопки). Добавить `IDI_APP` с ассоциацией в манифесте.

### 21. Сохранение положения и размера окна
Окно фиксировано (DS_SYSMODAL), нет resizable-стиля. Если убрать `DS_SYSMODAL` — сохранять координаты в реестре через `WM_WINDOWPOSCHANGED`.

### 22. Переименовать «Стандартная» → «Размеры»
Вкладка «Стандартная» (FileStandardInfo) содержит только размеры и счётчики ссылок. Название не отражает содержимое.

### 23. Убрать или скрыть чекбоксы атрибутов недоступные для текущего файла
Некоторые атрибуты взаимоисключающие (Normal несовместим с любым другим) или нечитаемые (Encrypted + Compressed одновременно невозможно). Добавить логику взаимного отключения.

## Критические баги

### 24. ~~Двойное нажатие кнопки Save не защищено~~ ✅ ИСПРАВЛЕНО
~~Быстрые повторные клики на «Сохранить» в Basic Info могут привести к двойному вызову `SetFileInformationByHandle`. Добавить флаг «в процессе сохранения» или отключать кнопку на время записи.~~

**Исправление:** `file_basic_info.c`, обработчик `IDC_FILEINFO_SAVE` — кнопка отключается (`EnableWindow(FALSE)`) в самом начале обработчика. `private_UpdateButtonState` в конце повторно включает её только если есть несохранённые изменения.

### 25. ~~Утечка дескриптора при ошибке открытия потока~~ ✅ НЕ ПОДТВЕРЖДЁН
~~В `file_stream_info.c` при некоторых ошибках `CloseHandle` не вызывается (ранний `return` после `common_ShowError`). Проверить все ветки выхода.~~

**Результат проверки:** все ветки ранних `return` в `private_LoadFileToStream` и `private_SaveStream` корректно закрывают дескрипторы: при сбое открытия второго хендла первый закрывается перед возвратом; при сбое I/O оба хендла закрываются в финальном блоке. Утечки отсутствуют.

### 26. ~~Переполнение буфера при длинном пути в MessageBox confirm~~ ✅ ИСПРАВЛЕНО
~~Функция `StringCbPrintf(szConfirm, ...)` использует буфер 2048 байт. Путь файла + имя потока могут суммарно превысить лимит. Использовать динамический буфер или `_vscwprintf`.~~

**Исправление:** `file_stream_info.c`, `private_LoadFileToStream` — статический буфер `TCHAR szConfirm[MAX_PATH*3]` заменён на `LPTSTR szConfirm = NULL`. Размер вычисляется через `_scwprintf()` по реальной длине входных строк, затем выделяется буфер через `HeapAlloc`, строка форматируется `swprintf_s`, после показа MessageBox буфер освобождается.
